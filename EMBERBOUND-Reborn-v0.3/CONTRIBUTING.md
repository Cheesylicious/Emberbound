# Beitragen

Bitte Änderungen klein und nachvollziehbar halten.

1. Neuen Branch anlegen.
2. Inhalte bevorzugt in `src/data.js` ergänzen.
3. Neue Regeln in `src/game.js` dokumentiert implementieren.
4. `npm test` ausführen.
5. Bei Grafikänderungen `docs/ART_BIBLE.md` beachten.
6. Im Pull Request Spielstand-Kompatibilität und getestete Geräte nennen.

Keine aus anderen Spielen extrahierten Grafiken, Klänge, Dialoge oder Daten
einreichen. Neue Assets benötigen eine dokumentierte Herkunft und Lizenz.
